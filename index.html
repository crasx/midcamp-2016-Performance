Purpose

The purpose of this talk is...

  • to avoid situations like that
  • to provide an understanding of how to support small to medium size organizations that have the opportunity to drive big traffic
  • to show you need to be intimate with your environment when dealing with performance and uptime
  • to show performance is an ongoing concern that can make or break you

What we will be covering

  • Ecommerce
  • How to create a holiday performance plan
  • How to test (tools) and optimize before the big day
  • What to test before the big day
  • Analyzing results and optimizing
  • Diagnosis checklist for the big day
  • Repairing in real time
  • Plan B

Some givens about performance

  • Minimize your site's footprint
  • Cache everything
    • Application / Drupal cache
    • PHP Cache
    • SQL cache
  • Optimize your frontend
    • Image optimization
    • CSS/JS Compression
    • Minimize external requests
  • Use gzip or some backend compression

Step 1: Make a plan

  • Start Early
  • Determine your goals and key metrics
  • Determine Budget
  • Balance capacity vs. performance vs. availability
  • Be aware of marketing's calendar
  • Define both desired user behaviors and high impact behhavior
  • Create a bottleneck list and define alternatives for points of failure
  • How will you scale and under what conditions?
  • Failing gracefully (aka "Plan C")

Step 2: Determine what to test

  • Common user flows / Critical paths
    • Browse → Add to cart → checkout
    • Login → browse → add to wishlist
  • Custom built pages
    • Build a wagon
    • Build a tesla
  • "Intense" processes
    • Migrate API
    • Search large data sets

Step 4: Identify problems.
Fix them!

Server Utilization

  • 100% CPU / RAM
  • Too much "stuff" on one server
    • Separate it - SQL - NFS - Cache
    • Use better caching
    • Scale horizontally
  • Try varnish Lots of setup time

Network Usage

  • Less common, more deadly
  • PHP - opcache
  • Monitor your caching
  • Delay external requests
  • CDN

SQL Overload

  • Cool area of study you could spend a whole career on
  • Magento promo codes suck
  • Solr lucene, Cassandra, searchapi
  • Master/Slave
  • Complex databases = complex solution
  • Know your limits
    • We hit 40 checkouts per minute

Global State

Set data-state="something" on a slide and "something" will be added as a class to the document element when the slide is open. This lets you apply broader style changes, like switching the page background.

State Events

Additionally custom events can be triggered on a per slide basis by binding to the data-state name.

	console.log( '"customevent" has fired' );
